# 📚 Библиотека АБИС — Конфигурация 1С:Предприятие 8.3

Курсовой проект РТУ МИРЭА, дисциплина «Основы проектной деятельности» (ОПД), **Кейс 8: Библиотека — выдача и просрочки**.
Подсистема автоматизирует учёт книг, читателей, выдач/возвратов и формирование отчёта «Просрочки».

---

## 1. Структура каталога

```
1c/
├── 1Cv8.dt                                 # Дамп информационной базы (канонический артефакт)
├── ExchangeFormats/                        # Шаблоны CSV для двустороннего обмена с FastAPI
│   ├── books_template.csv
│   ├── readers_template.csv
│   └── loans_template.csv
└── README.md                               # Этот файл
```

Внутри `1Cv8.dt` упакованы все объекты конфигурации (справочники, документы, регистр, отчёты, общие модули и роли). Распаковка/выгрузка в XML делается стандартными средствами Конфигуратора при необходимости — отдельно в репозитории не дублируется, чтобы избежать рассинхрона между текстовой и бинарной версией.

## 3. Быстрый старт

### Загрузка информационной базы

1. Запустите 1С — кнопка **«Добавить»** → «Создание новой информационной базы».
2. Выберите «Создание информационной базы без конфигурации…» → укажите каталог.
3. Откройте созданную ИБ в режиме **«Конфигуратор»**.
4. Меню **«Администрирование» → «Загрузить информационную базу…»** → укажите файл `1c/1Cv8.dt`.
5. Подтвердите перезапуск Конфигуратора.
6. Запустите режим **«Предприятие»** — на стартовой странице появится **«Рабочее место библиотекаря»**.

### Демо-учётные записи

| Логин              | Роль          |
|--------------------|---------------|
| Администратор      | полный доступ |
| Библиотекарь       | выдачи, отчёты, импорт/экспорт |
| Читатель           | каталог, личные выдачи |

### Первичная загрузка данных

В сеансе под ролью «Администратор» из подсистемы «Импорт CSV» по очереди:
- `ExchangeFormats/books_template.csv`   — справочник книг;
- `ExchangeFormats/readers_template.csv` — справочник читателей;
- `ExchangeFormats/loans_template.csv`   — исторические выдачи (опционально).

Импорт **идемпотентен**: повторная загрузка не создаёт дубли. Ключи: `book_code`, `reader_id`, `loan_id`.

## 4. Объекты конфигурации (внутри `1Cv8.dt`)

### Справочники
- **Книги** — `book_code`, `Наименование`, `Автор книги`, `Год издания книги`, `УДК`, `Статус`.
- **Читатели** — `reader_id`, `Фамилия и инициалы`, `Группа`, `Категория`, `Email`, `Телефон`, `ЗаблокированВыдачи`.

### Документы
- **ВыдачаКниги** — `Книга`, `Читатель`, `ДатаВыдачи`, `ПлановаяДатаВозврата`. «+» по регистру «Выдачи».
- **ВозвратКниги** — `ДокументВыдачи`, `ДатаВозврата`, `ДнейПросрочки`, `СостояниеКниги`. «−» по регистру «Выдачи».

### Регистр накопления
- **Выдачи** (обороты): Книга, Читатель, Количество, ДокументВыдачи, ПлановаяДатаВозврата.

### Отчёты (СКД)
- **Просрочки** — обязательный по ТЗ: `reader_id`, `Фамилия и инициалы`, `book_code`, `title`, `days_overdue` + экспорт в CSV.
- **ОборачиваемостьФонда** — KPI: доля активных, доля просрочек, средняя/медианная просрочка, ТОП-N.

## 5. Бизнес-правила (из ТЗ)

| Код   | Правило                                                     |
|-------|-------------------------------------------------------------|
| BR-01 | Одна активная выдача на один `book_code`                    |
| BR-02 | `issue_date ≤ due_date`; `return_date ≥ issue_date`         |
| BR-03 | Ссылочная целостность: `reader_id` существует в справочнике |
| BR-04 | Ссылочная целостность: `book_code` существует в справочнике |
| BR-05 | Возврат невозможен без исходного документа выдачи           |
| BR-06 | Заблокированный читатель не может получать новые выдачи     |

Контракты валидируются в обработчиках проведения соответствующих документов.

## 6. Интеграция с FastAPI-бэкендом

Обмен идёт через CSV-файлы по контракту, общему для обеих подсистем.

### 6.1 Из 1С → в backend (импорт справочников/выдач)

| Эндпоинт FastAPI                              | Что отправляем           | CSV-шаблон                |
|-----------------------------------------------|--------------------------|---------------------------|
| `POST /api/import-export/books/import`        | Справочник книг          | `books_template.csv`      |
| `POST /api/import-export/readers/import`      | Справочник читателей     | `readers_template.csv`    |
| `POST /api/import-export/loans/import`        | Журнал выдач/возвратов   | `loans_template.csv`      |

### 6.2 Из backend → в 1С (выгрузка обновлений)

| Эндпоинт FastAPI                              | Что получаем             |
|-----------------------------------------------|--------------------------|
| `GET /api/import-export/books/export`         | Каталог книг (CSV)       |
| `GET /api/import-export/readers/export`       | Читатели (CSV)           |
| `GET /api/import-export/loans/export`         | Выдачи и возвраты (CSV)  |
| `GET /api/reports/overdue/export`             | Отчёт «Просрочки» (CSV)  |

Кодировка CSV — UTF-8 (с поддержкой cp1251 на стороне backend), разделитель — `;`.
Маппинг полей — `docs/05-data-and-integration/DOC-INT-003-field-mapping-1c.md`.
