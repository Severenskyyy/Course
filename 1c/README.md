# 📚 Библиотека АБИС — Конфигурация 1С:Предприятие 8.3

Курсовой проект РТУ МИРЭА, дисциплина «Основы проектной деятельности» (ОПД), **Кейс 8: Библиотека — выдача и просрочки**.
Подсистема автоматизирует учёт книг, читателей, выдач/возвратов и формирование отчёта «Просрочки».

## 3. Быстрый старт

1. Создайте пустую информационную базу.
2. Откройте её в конфигураторе (`Конфигуратор → Открыть конфигурацию`).
3. Загрузите выгрузку: `Конфигурация → Загрузить файлы` → укажите папку `Configuration/`.
4. Выполните «Обновление конфигурации базы данных» (F7).
5. Запустите режим «Предприятие».
6. Откройте **Рабочее место библиотекаря** со стартовой страницы.

### Первичная загрузка данных

Из меню «Администратор → Импорт CSV» последовательно:
- `books_template.csv`   — справочник книг;
- `readers_template.csv` — справочник читателей;
- `loans_template.csv`   — исторические выдачи (опционально).

Импорт **идемпотентен** — повторная загрузка не создаёт дублей (ключи: `book_code`, `reader_id`, `loan_id`).

## 4. Объекты конфигурации

### Справочники
- **Книги** — Код (book_code), Наименование (title), Автор, ГодИздания, УДК, Статус.
- **Читатели** — Код (reader_id), ФИО, Группа, Категория, Email, Телефон, ЗаблокированВыдачи.

### Документы
- **ВыдачаКниги** — Книга, Читатель, ДатаВыдачи, ПлановаяДатаВозврата. «+» по регистру «Выдачи».
- **ВозвратКниги** — ДокументВыдачи, ДатаВозврата, ДнейПросрочки, СостояниеКниги. «−» по регистру «Выдачи».

### Регистр накопления
- **Выдачи** (обороты): Книга, Читатель, Количество, ДокументВыдачи, ПлановаяДатаВозврата.

### Отчёты
- **Просрочки** — обязательный по ТЗ: reader_id, fio, book_code, title, days_overdue + экспорт CSV.
- **Оборачиваемость фонда** — KPI: доля активных, доля просрочек, средняя/медианная просрочка, ТОП-N.

## 6. Бизнес-правила (из ТЗ)

| Код   | Правило                                                        | Модуль                                 |
|-------|----------------------------------------------------------------|----------------------------------------|
| BR-01 | Одна активная выдача на один book_code                         | `Document.BookIssue.ObjectModule`      |
| BR-02 | issue_date ≤ due_date; return_date ≥ issue_date                | `ОбщегоНазначения.ПроверитьДатыВыдачи` |
| BR-03 | Ссылочная целостность: reader_id существует                    | Проверка при проведении                |
| BR-04 | Ссылочная целостность: book_code существует                    | Проверка при проведении                |
| BR-05 | Возврат невозможен без исходного документа выдачи              | `Document.BookReturn.ObjectModule`     |
| BR-06 | Заблокированный читатель не может получить новые выдачи        | Обработчик проведения выдачи           |

## 7. Интеграция с FastAPI-бэкендом

Двусторонний обмен JSON через `Modules/DataExchange.bsl`:

- **Выгрузка в бэкенд:**
  - `POST /api/import-export/books`   — справочник книг;
  - `POST /api/import-export/readers` — справочник читателей;
  - `POST /api/import-export/loans`   — движения регистра выдач.
- **Загрузка из бэкенда:**
  - `GET  /api/books`                    — обновление каталога;
  - `GET  /api/readers`                  — обновление читателей;
  - `GET  /api/reports/overdue/export`   — CSV-отчёт о просрочках.

Маппинг полей — `docs/05-data-and-integration/DOC-INT-003-field-mapping-1c.md`.
