# Кейс 8 — Библиотека: выдача и просрочки

Исходный текст кейса из документа «Кейсы для дисциплины «Основы проектной деятельности»».
Копия сохранена в репозитории для автономного доступа команды и проверяющих.

---

**Отрасль:** образование / госсектор
**Бизнес-цель:** снизить просрочки возврата и повысить оборот фонда.
**KPI (пример):** доля просроченных выдач ≤ целевого порога на наборе T1.

## Расширенное описание

Смоделируйте учёт каталога, читателей, выдач/возвратов и просрочек.
Реализуйте статусы и отчёт «Просрочки».

## Данные (синтетика; схема)

Таблицы:
- `books` (book_code, title, author, year, udk)
- `readers` (reader_id, fio, group)
- `loans` (loan_id, reader_id, book_code, issue_date, due_date, return_date)

Объёмы: ≥ 100 выдач.

Валидации: один экземпляр — одна активная выдача; `issue_date ≤ due_date ≤ return_date?`;
корректная просрочка: `return_date=null и today > due_date`, или `return_date > due_date`.

## 1) Общие положения

Система моделирует базовый АБИС-контур: каталог книг, читатели, учёт выдач/возвратов, определение просрочек и отчётность.
Проект учебный: без интеграций с внешними АБИС, работа с CSV и формами 1С.

## 2) Основные бизнес-правила

1. **Каталог и экземпляры** — `book_code` трактуем как экземпляр (0/1 активная выдача).
2. **Жизненный цикл выдачи** — создание с `issue_date`/`due_date`, возврат — `return_date`, просрочка по ТЗ.
3. **Ограничения и проверки** — одна активная выдача, корректность дат, ссылочная целостность.
4. **Политики библиотеки** — штрафы/продления/бронирования опциональны (Could).

## 3) Нефункциональные требования

- Производительность: отчёт по ≥ 100 записям интерактивно.
- Надёжность: идемпотентный импорт CSV.
- Трассируемость: журнал изменений.
- Доступы: роли «библиотекарь», «администратор», «читатель».

## 4) Отчётность

- **Отчёт «Просрочки»**: `reader_id, fio, book_code, title, days_overdue`.
- Показатели: доля просроченных, средняя/медианная, ТОП-N.
- Разрезы: период, группа, автор/UDK.
- Экспорт CSV обязателен.

## 5) Интеграция

- CSV-импорт/экспорт.
- В перспективе: уведомления, ЛК читателя, выгрузка BI (вне скоупа MVP).

## 6) Компоненты 1С

- 1С:Предприятие 8.3 — формы, документы, регистры, отчёты.
- 1С:Документооборот 8 (опц.) — регламенты по просрочкам.

## 7) Датасеты

Синтетические (T1), ≥ 100 выдач. Справочные каталоги из открытых источников — только как дополнение.

## 8) Критерии приёмки

1. Корректные статусы выдач (в т.ч. просрочка).
2. Отчёт «Просрочки» (reader_id, book_code, days_overdue).
3. Экспорт отчёта в CSV.
4. README со шагами проверки.

## 9) Роли / задачи

- **P1 Платформенные решения** (1С) — формы/отчёт, импорт/экспорт CSV.
- **P2 Фуллстек** — UI-прототип каталога/выдач.
- **P3 Информатизация организаций** — BPMN, ERD, user stories.
- **P4 Цифровая трансформация** — KPI просрочек/оборачиваемости, план валидации.
- **P5 ИСУПР** — регистр «Выдачи», тест-наборы и валидаторы.

*Ограничения:* без интеграций с АБИС; только синтетика.
