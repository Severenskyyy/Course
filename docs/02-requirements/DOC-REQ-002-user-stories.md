## User Stories

### US-01: Поиск книг в каталоге
**Как** библиотекарь, **я хочу** искать книги по названию, автору и коду, **чтобы** быстро найти нужный экземпляр.

**Критерии приёмки:**
- Поиск по подстроке в title, author, book_code
- Фильтрация по автору, УДК, наличию
- Результаты отображаются мгновенно (< 1 сек)

### US-02: Добавление книги
**Как** библиотекарь, **я хочу** добавить новую книгу в каталог, **чтобы** учитывать поступления в фонд.

**Критерии приёмки:**
- Форма: book_code, title, author, year, udk (обязательные)
- Проверка уникальности book_code
- Книга сразу появляется в каталоге со статусом «Доступна»

### US-05: Оформление выдачи
**Как** библиотекарь, **я хочу** оформить выдачу книги читателю, **чтобы** зафиксировать факт выдачи и срок возврата.

**Критерии приёмки:**
- Выбор читателя и книги
- Проверка: книга не имеет активной выдачи
- Фиксация issue_date (сегодня) и due_date (выбирается)
- Книга помечается как «Выдана»

### US-06: Оформление возврата
**Как** библиотекарь, **я хочу** оформить возврат книги, **чтобы** закрыть выдачу и освободить экземпляр.

**Критерии приёмки:**
- return_date = текущая дата
- Автоматический расчёт days_overdue
- Книга помечается как «Доступна»

### US-08: Отчёт «Просрочки»
**Как** библиотекарь, **я хочу** видеть список просроченных выдач, **чтобы** связаться с должниками.

**Критерии приёмки:**
- Поля: reader_id, fio, book_code, title, days_overdue
- Фильтры: период, группа, автор/УДК
- Экспорт в CSV
- Показатели: доля просрочек (%), средняя/медианная просрочка

### US-11: Импорт данных из CSV
**Как** администратор, **я хочу** загрузить данные из CSV, **чтобы** массово наполнить систему.

**Критерии приёмки:**
- Поддержка books.csv, readers.csv, loans.csv
- Идемпотентность: повторная загрузка не создаёт дублей
- Отчёт: добавлено / пропущено / ошибок

## MoSCoW

| Приоритет | Требования |
|---|---|
| **Must** | CRUD книг, CRUD читателей, выдача/возврат, отчёт Просрочки, экспорт CSV, импорт CSV |
| **Should** | JWT авторизация (3 роли), KPI-дашборд, поиск с фильтрами |
| **Could** | Бронирование, уведомления, аудит-лог |
| **Won't** | Email/SMS рассылки, REST API вместо CSV для 1С, мобильное приложение |
