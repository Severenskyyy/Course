## 5. Прослеживаемость на User Stories

Каждое M-требование имеет соответствующую User Story в `DOC-REQ-002`; каждая M-сторiа покрыта acceptance-тестом в `DOC-TST-001`. Полная матрица — в `DOC-SPC-001` (Финальное ТЗ).

## 6. Процесс изменения

1. Issue в GitHub с тегом `moscow-change`.
2. Обсуждение на следующей команде-встрече.
3. Согласование с куратором (если затронуты Must).
4. Обновление этой матрицы, инкремент версии.
