## 2. Назначение и цели создания системы

### 2.1. Проблема, решаемая проектом
В библиотеках часто теряется контроль за возвратом книг. Отсутствие автоматизации приводит к росту задолженностей, утере экземпляров и снижению оборачиваемости фонда. Учёт ведётся в бумажных журналах. Библиотекарь не знает о просрочках до визита читателя. Данные в 1С переносятся вручную.

### 2.2. Целевая аудитория
- **Библиотекарь** — оформляет выдачи/возвраты, формирует отчёты.
- **Администратор** — управляет пользователями, импорт/экспорт данных.
- **Читатель (студент)** — просматривает каталог, отслеживает сроки.

### 2.3. Назначение системы
Система АБИС предназначена для автоматизации учёта каталога книг, управления читателями, регистрации выдач/возвратов, определения просрочек и формирования отчётности с интеграцией с 1С через CSV.

### 2.4. Цель создания системы
Разработка MVP: снижение доли просроченных выдач до ≤ 15%, автоматический расчёт дней просрочки, формирование отчёта с экспортом CSV и идемпотентный обмен с 1С.

### 2.5. Задачи проекта
1. Спроектировать модель данных (ERD ≥ 5 сущностей) и бизнес-процесс (BPMN).
2. Реализовать REST API для CRUD операций с книгами, читателями и выдачами.
3. Реализовать формы и отчёт в 1С:Предприятие 8.3 с CSV-обменом.
4. Разработать веб-интерфейс (React) с JWT-авторизацией (3 роли).
5. Обеспечить формирование отчёта «Просрочки» с фильтрами и экспортом CSV.

### 2.6. Границы MVP
1. CRUD каталога книг (поиск, фильтрация).
2. CRUD читателей (поиск по ФИО/группе).
3. Регистрация выдачи с проверкой доступности.
4. Оформление возврата с расчётом days_overdue.
5. Отчёт «Просрочки» с фильтрами и экспортом CSV.

### 2.7. Что не входит в MVP
1. Интеграция с внешними АБИС (Ирбис, Руслан).
2. Мобильное приложение.
3. Оплата штрафов.
4. Сканирование штрих-кодов.
5. Email/SMS-рассылка.

## 3. Характеристика объекта автоматизации

### 3.1. Краткая характеристика предметной области
Учебная библиотека университета. Фонд — несколько тысяч экземпляров. Читатели — студенты по группам. Срок выдачи 7–21 день.

### 3.2. Текущий способ решения задачи
Бумажные журналы или Excel. Каталог — разрозненные файлы. Контроль просрочек — визуальный. Данные в 1С — вручную.

### 3.3. Ограничения предметной области
1. Синтетические данные (≥ 100 строк на таблицу).
2. Без внешних интеграций и ПДн.
3. Срок — 8 недель, команда — 4 человека.
4. 1С — учебная лицензия.

## 5. Описание основного бизнес-процесса

### 5.1. Наименование
Выдача и возврат книги в библиотеке

### 5.2. Краткое описание
Читатель запрашивает книгу → библиотекарь проверяет наличие → оформляет выдачу → по истечении срока книга возвращается → система рассчитывает просрочку → библиотекарь формирует отчёт.

### 5.3. Участники
1. Читатель
2. Библиотекарь
3. Система АБИС

### 5.4. Шаги процесса

| № | Роль | Действие | Входные данные | Результат |
|---|---|---|---|---|
| 1 | Читатель | Запрос на выдачу | ФИО, книга | Обращение |
| 2 | Библиотекарь | Поиск книги/читателя | reader_id, book_code | Найдены |
| 3 | Система | Проверка: экземпляр свободен | book_code | Да/Нет |
| 4 | Библиотекарь | Оформление выдачи | due_date | loan создан |
| 5 | Система | Запись loan + даты | Данные | loan_id |
| 6 | Читатель | Возврат книги | Экземпляр | Передан |
| 7 | Библиотекарь | Оформление возврата | loan_id | return_date |
| 8 | Система | Расчёт days_overdue | Даты | Просрочка |
| 9 | Библиотекарь | Отчёт Просрочки | Фильтры | Таблица+KPI |
| 10 | Библиотекарь | Экспорт CSV | — | overdue.csv |

## 7. Требования к данным

### 7.1. Сущности (7 шт.)
books, readers, loans, users, audit_log, reservations, notifications

### 7.2. Описание

| Сущность | Назначение | Атрибуты | PK |
|---|---|---|---|
| books | Каталог | book_code, title, author, year, udk, is_available | book_code |
| readers | Читатели | reader_id, fio, group_name, email, phone | reader_id |
| loans | Выдачи | loan_id, reader_id(FK), book_code(FK), issue_date, due_date, return_date | loan_id |
| users | Авторизация | user_id, username, password_hash, role, full_name, reader_id(FK) | user_id |
| audit_log | Аудит | log_id, user_id(FK), entity_type, entity_id, action, created_at | log_id |
| reservations | Бронирование | reservation_id, reader_id(FK), book_code(FK), expires_at | reservation_id |
| notifications | Уведомления | notification_id, reader_id(FK), loan_id(FK), type, message, is_read | notification_id |

### 7.3. Связи
books 1:M loans, readers 1:M loans, users 1:M audit_log, readers 1:M reservations, readers 1:M notifications, users 1:1 readers

### 7.4. Ограничения
1. PK — уникальные, не пустые.
2. title, author, fio, group — обязательные.
3. year: 1000–2100.
4. issue_date ≤ due_date; return_date ≥ issue_date.
5. FK: reader_id → readers, book_code → books.
6. 1 book_code = max 1 активная выдача.
7. Идемпотентность по PK.
8. role ∈ {admin, librarian, reader}.

## 8. Нефункциональные требования

### 8.1. Интерфейс
Простой, единообразный. Основные действия — не более 3 шагов. Навигация: Navbar + Sidebar.

### 8.2. Производительность
Отчёт < 3 сек на 600 записей. Корректная работа при 1–5 пользователях.

### 8.3. Надёжность
SQLite — файловая БД, данные сохраняются. Идемпотентный импорт.

### 8.4. Безопасность
JWT-авторизация. Pydantic-валидация. bcrypt-хеши паролей.

### 8.5. Удобство
Понятно без обучения. Ошибки на русском. Обязательные поля помечены «*».

## 10. Критерии приёмки MVP

### 10.1. Общие
1. Импорт T1 (≥200 книг, ≥120 читателей, ≥400 выдач) успешен.
2. Отчёт Просрочки корректен (days_overdue > 0).
3. Блокировка двойной выдачи (409).
4. Экспорт CSV: reader_id, book_code, days_overdue.
5. Идемпотентность: 0 дублей при повторной загрузке.

### 10.2. Проверка FR

| Код | Способ проверки | Ожидаемый результат |
|---|---|---|
| FR-01 | Добавить книгу, найти по автору | Книга в каталоге |
| FR-02 | Добавить читателя, найти по ФИО | Читатель в списке |
| FR-03 | Выдать книгу с активной выдачей | Ошибка 409 |
| FR-04 | Возврат просроченной книги | days_overdue > 0 |
| FR-05 | Открыть отчёт | Таблица с 5 полями |
| FR-06 | Фильтр по группе ИС-21 | Только ИС-21 |
| FR-07 | Экспорт CSV | Файл скачан |
| FR-08 | Загрузить T1 дважды | Записи не удвоились |
| FR-09 | Войти как reader, создать книгу | 403 Forbidden |
| FR-10 | Дашборд | KPI отображается |
