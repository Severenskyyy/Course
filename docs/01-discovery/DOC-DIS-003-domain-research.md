## 1. Цель документа

Зафиксировать результаты анализа предметной области «выдача книг и учёт просрочек» в библиотеке, чтобы последующие требования, модель данных и процессы опирались на единую картину мира.

## 2. Контекст отрасли

Библиотечное дело в образовательных учреждениях регулируется ГОСТ 7.0.103-2018 «Библиотечно-информационное обслуживание». Ключевые функции библиотеки учебного заведения:

1. Комплектование и учёт фонда (каталог).
2. Обслуживание читателей (регистрация, абонемент, читальный зал).
3. Учёт выдач и возвратов.
4. Контроль задолженности и просрочек.
5. Отчётность перед руководством и ФОИВ.

## 3. Типовой бизнес-процесс «выдача–возврат»

```
Поиск книги  →  Проверка читателя  →  Выдача  →  (срок)  →  Возврат
                        │                                     │
                        └── проверка долгов                    └── расчёт просрочки
```

- Стандартный срок выдачи: 14 дней для учебной литературы, 30 дней для научной.
- Просрочка начинается со дня, следующего за плановой датой возврата.
- Повторная выдача той же книги до возврата запрещена.

## 4. Ключевые концепты предметной области

| Термин        | Определение                                                            | Синоним в коде      |
|---------------|------------------------------------------------------------------------|---------------------|
| Экземпляр     | Физическая копия книги с уникальным инвентарным номером                | `book_code`         |
| Читатель      | Зарегистрированный пользователь библиотеки                             | `reader_id`         |
| Выдача        | Факт передачи экземпляра читателю на определённый срок                 | `loan`              |
| Возврат       | Факт возвращения экземпляра в фонд                                     | `return`            |
| Просрочка     | Ситуация, когда возврат не произошёл в срок (`today > due_date`)       | `overdue`           |
| Абонемент     | Подсистема выдачи на дом                                               | —                   |
| UDK (УДК)     | Универсальная десятичная классификация, систематизирует фонд           | `udk`               |

## 5. Аналоги и best-practices

Изучены открытые сведения о следующих системах:

1. **ИРБИС64** — отечественная промышленная АБИС, используется в большинстве вузов. Взяты идеи: двойной учёт (экземпляр + издание), статусы выдач, журнал аудита.
2. **Koha** (open-source) — международный стандарт. Взяты идеи: `item status` enum, гибкие правила срока выдачи по категориям читателей.
3. **1С:Библиотека** — референс для 1С-подсистемы. Взято: структура справочников, регистр накопления «Выдачи».

## 6. Границы учебного проекта (MVP)

В учебном проекте намеренно **не реализуются**:
- Бронирование и очередь на книгу.
- Штрафы и денежные расчёты.
- Продление срока выдачи.
- Многотиражный учёт (один `book_code` = один экземпляр).
- Интеграция с внешними АБИС и каталогами типа РГБ.
- ЛК читателя с мобильной нотификацией.

Эти элементы зафиксированы в MoSCoW как Could/Won’t и документированы как возможные расширения.

## 7. Нормативно-правовой контекст

- ФЗ «О библиотечном деле» № 78-ФЗ.
- ГОСТ 7.0.103-2018, ГОСТ 7.60-2003 (виды изданий).
- Устав РТУ МИРЭА (локальный акт): «Правила пользования библиотекой».

Для учебного проекта детальная юридическая привязка не требуется, но в ТЗ явно указано, что продукт — учебный макет, не предназначенный к промышленной эксплуатации.

## 8. Выводы для проекта

1. Основная ценность — сокращение просрочек и ускорение оборота фонда → KPI на дашборде.
2. Главная сущность — `loan` (выдача), вокруг неё строится модель данных.
3. Правило «один `book_code` — одна активная выдача» определяет ключевую бизнес-валидацию.
4. Отчёт «Просрочки» — обязательный артефакт, именно он формализует ценность для заказчика.
