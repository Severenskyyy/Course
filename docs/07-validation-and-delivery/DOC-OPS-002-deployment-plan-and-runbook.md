## 1. Цель

Пошаговая инструкция для развёртывания учебного стенда АБИС на машине для демонстрации на защите. Ориентирована на Windows 11 и macOS/Linux.

## 2. Требования к окружению

| Компонент          | Минимум                                  | Проверка                                 |
|--------------------|------------------------------------------|------------------------------------------|
| Python             | 3.11                                     | `python --version`                       |
| Node.js            | 20.x                                     | `node --version`                         |
| 1С:Предприятие     | 8.3.20 учебная редакция                  | По меню «Справка → О программе»          |
| Git                | 2.40+                                    | `git --version`                          |
| Свободного места   | 500 МБ                                   | `df -h` / проводник                      |
| RAM                | 4 ГБ                                     | Диспетчер задач                          |

## 3. Порядок развёртывания

### Шаг 1. Получение исходников

```bash
git clone https://github.com/Severenskyyy/Course.git
cd "Course/Наш проект"
```

### Шаг 2. Запуск Backend (FastAPI)

```bash
cd backend
python -m venv .venv
source .venv/bin/activate        # Windows: .venv\Scripts\activate
pip install -r requirements.txt

# Инициализация БД и загрузка демо-данных
python -m app.cli init_db
python -m app.cli load_demo data/T1

# Запуск сервера
uvicorn app.main:app --reload --port 8000
```

Проверка: в браузере `http://localhost:8000/docs` должна открыться Swagger UI.

### Шаг 3. Запуск Frontend (React)

В новом терминале, из корня проекта:

```bash
npm install
npm run dev
```

Проверка: `http://localhost:5173` — экран логина АБИС. Учётка для демо: `librarian / library2026`.

### Шаг 4. Развёртывание 1С-конфигурации

1. Открыть 1С:Предприятие → Администрирование → Создать → «Создание новой информационной базы».
2. Выбрать пустую информационную базу.
3. Открыть конфигуратор → Конфигурация → Загрузить файлы... → указать папку `1c/Configuration/`.
4. F7 (Обновить конфигурацию базы данных).
5. Режим «Предприятие» → открыть «Рабочее место библиотекаря».
6. Администратор → Импорт CSV → последовательно `books_template.csv`, `readers_template.csv`, `loans_template.csv`.

### Шаг 5. Проверка демо-сценария

1. В React-UI (librarian): открыть `Catalog` — список ≥ 50 книг.
2. Открыть `Loans → Новая выдача`: выбрать читателя, книгу, срок — нажать «Выдать».
3. Открыть `Overdue` → «Экспорт CSV»: проверить, что файл скачан.
4. В 1С открыть «Рабочее место библиотекаря» → проверить те же операции.

## 4. Smoke-тест (runbook)

| Шаг | Ожидание                                                                 | Результат |
|-----|--------------------------------------------------------------------------|-----------|
| 1   | `curl -s http://localhost:8000/health` → `200 {"status":"ok"}`           |           |
| 2   | Логин `admin / admin2026` → токен в ответе                               |           |
| 3   | `GET /api/books` с токеном → список ≥ 100                                |           |
| 4   | `GET /api/reports/overdue` → объект с `items` ≥ 1 на T1                  |           |
| 5   | Фронт `Login` → пароль → редирект на `Home`                              |           |
| 6   | Экспорт CSV → скачан файл `overdue_YYYYMMDD.csv`                          |           |

## 5. Что делать, если не работает

| Симптом                                    | Действие                                                   |
|--------------------------------------------|------------------------------------------------------------|
| Python 3.11 не установлен                  | Установить с python.org, добавить в PATH                   |
| `ModuleNotFoundError: fastapi`             | Активировать venv и повторить `pip install -r requirements.txt` |
| Порт 8000 занят                            | `uvicorn ... --port 8001`, во фронте править `api.ts`      |
| В 1С «Не удалось загрузить конфигурацию»   | Проверить, что лицензия «ПРОФ»; режим совместимости 8.3.20 |
| CSV-импорт падает «неверная кодировка»     | Сохранить файл с BOM UTF-8                                 |
| Ошибка JWT «Signature verification failed` | Проверить `.env` — один `SECRET_KEY` в BE и проде          |

## 7. План отката (rollback)

Если на защите обнаружена критическая ошибка:
1. Останавливаем реальный Backend, запускаем мок-сервер `data/T1/mock_server.py`.
2. Фронт продолжит работать на мок-данных.
3. Если проблема с 1С — демонстрируем XML-выгрузку конфигурации и pre-recorded видео демо.
