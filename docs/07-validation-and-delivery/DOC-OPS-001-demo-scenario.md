# Сценарий демо — АБИС (Кейс 8)

## Подготовка
- Запустить backend: `cd backend && uvicorn app.main:app --reload --port 8000`
- Запустить frontend: `cd . && npm run dev`
- Убедиться, что data/T1/ содержит books.csv, readers.csv, loans.csv

## Шаги демо

### Шаг 1. Авторизация (30 сек)
**Вход:** логин admin / admin123  
**Действие:** ввести логин и пароль, нажать «Войти»  
**Результат:** отображается главная страница с дашбордом KPI  
**Скрин:** Главная с виджетами KPI

### Шаг 2. Импорт данных CSV (1 мин)
**Вход:** файлы data/T1/books.csv, readers.csv, loans.csv  
**Действие:** перейти на страницу «Интеграция», загрузить файлы по очереди  
**Результат:** система сообщает: «Импортировано: 250 книг, 150 читателей, 600 выдач. Дубли: 0»  
**Проверка:** повторная загрузка — «Пропущено: 250, Добавлено: 0» (идемпотентность)

### Шаг 3. Просмотр каталога (30 сек)
**Действие:** перейти на страницу «Каталог», применить фильтр по автору «Пушкин»  
**Результат:** отображаются только книги Пушкина  
**Скрин:** Каталог с фильтром

### Шаг 4. Оформление выдачи (1 мин)
**Вход:** читатель RD0001, книга BK0050  
**Действие:** перейти на «Выдачи», нажать «Новая выдача», выбрать читателя и книгу, указать due_date через 14 дней  
**Результат:** выдача создана, книга отмечена как выданная  
**Проверка:** попытка повторной выдачи BK0050 → ошибка «Книга уже выдана»

### Шаг 5. Оформление возврата (30 сек)
**Действие:** найти активную выдачу в списке, нажать «Вернуть»  
**Результат:** return_date зафиксирована, days_overdue рассчитан автоматически  

### Шаг 6. Отчёт «Просрочки» (1 мин)
**Действие:** перейти на «Просрочки», задать фильтр по группе «ИС-21»  
**Результат:** отображается таблица с полями reader_id, fio, book_code, title, days_overdue  
**KPI:** доля просроченных выдач — отображается в процентах  

### Шаг 7. Экспорт CSV (30 сек)
**Действие:** нажать «Экспорт CSV» на странице просрочек  
**Результат:** скачан файл overdue_export.csv  
**Проверка:** сравнить структуру с data/golden/overdue.csv

## Общее время: 5 минут

## Откат для повторного прогона
Удалить файл backend/library.db — при следующем запуске данные будут пересозданы из демо-сида.
