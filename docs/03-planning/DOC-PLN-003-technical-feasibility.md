## 1. Цель

## 4. Ключевые технические риски и компенсации

1. **Учебная 1С-лицензия**. Получение занимает время → заложен буфер 1 неделя. При критической задержке — fallback на макет XML выгрузки (не исполняется, но демонстрирует структуру).
2. **Idempotency CSV-импорта**. Решение: натуральный ключ (`book_code`, `reader_id`, `loan_id`) + UPSERT.
3. **Производительность отчёта на T1 (≥100 записей)**. Решение: индексы на `loans.due_date`, `loans.return_date`.
4. **Синхронизация JSON-контракта между FastAPI и React**. Решение: один OpenAPI-файл = источник правды, pydantic-модели идентичны TS-типам.

## 5. PoC-проверки (что проверили экспериментально на неделях 1–2)

1. FastAPI + SQLite на 100 тестовых записях отчёта: ответ < 100 мс. ✅
2. Импорт CSV через pandas + idempotent upsert: 600 строк за 2.3 с. ✅
3. React + Vite single-file build (`index.html`): размер < 500 кБ. ✅
4. 1С:Предприятие учебная редакция запуск под Windows 11: ✅ (после установки лицензии).

## 6. Вывод

MVP реализуем в срок. Главный риск — задержка с 1С-лицензией — снят получением учебной базы на неделе 2. Команда обладает достаточной компетенцией по всем компонентам.
