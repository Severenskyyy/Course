# Перечень сущностей и Data Dictionary — АБИС

Кейс 8: Библиотека — выдача и просрочки | Версия 1.0 | Дата: 12.04.2026

## Перечень сущностей (≥5)

| # | Сущность | Описание | Кол-во строк (T1) |
|---|----------|----------|-------------------|
| 1 | **books** | Каталог экземпляров книг | 200–300 |
| 2 | **readers** | Читатели библиотеки | 120–200 |
| 3 | **loans** | Журнал выдач и возвратов | 400–800 |
| 4 | **users** | Пользователи системы (авторизация) | 10–20 |
| 5 | **audit_log** | Журнал аудита операций | ≥100 |

## Data Dictionary

### books (Каталог книг)

| Поле | Тип | PK/FK | Обязат. | Домен/Ограничения | Описание |
|------|-----|-------|---------|-------------------|----------|
| book_code | VARCHAR(20) | PK | Да | Уникальный, не пустой | Код экземпляра книги |
| title | VARCHAR(255) | — | Да | Не пустой | Название книги |
| author | VARCHAR(255) | — | Да | Не пустой | Автор |
| year | INTEGER | — | Нет | 1000–2026 | Год издания |
| udk | VARCHAR(50) | — | Нет | Формат УДК | Классификационный код |

### readers (Читатели)

| Поле | Тип | PK/FK | Обязат. | Домен/Ограничения | Описание |
|------|-----|-------|---------|-------------------|----------|
| reader_id | VARCHAR(20) | PK | Да | Уникальный, не пустой | Идентификатор читателя |
| fio | VARCHAR(255) | — | Да | Не пустой | ФИО читателя |
| group | VARCHAR(100) | — | Да | Не пустой | Группа/подразделение |

### loans (Журнал выдач)

| Поле | Тип | PK/FK | Обязат. | Домен/Ограничения | Описание |
|------|-----|-------|---------|-------------------|----------|
| loan_id | VARCHAR(20) | PK | Да | Уникальный, не пустой | ID выдачи |
| reader_id | VARCHAR(20) | FK → readers | Да | Существует в readers | Читатель |
| book_code | VARCHAR(20) | FK → books | Да | Существует в books | Экземпляр |
| issue_date | DATE | — | Да | <= due_date | Дата выдачи |
| due_date | DATE | — | Да | >= issue_date | Плановая дата возврата |
| return_date | DATE (NULL) | — | Нет | >= issue_date; если задана >= due_date — просрочка | Факт. дата возврата |

### users (Пользователи)

| Поле | Тип | PK/FK | Обязат. | Домен/Ограничения | Описание |
|------|-----|-------|---------|-------------------|----------|
| id | INTEGER | PK | Да | AUTO_INCREMENT | ID пользователя |
| username | VARCHAR(100) | — | Да | Уникальный | Логин |
| password_hash | VARCHAR(255) | — | Да | — | Хэш пароля |
| role | VARCHAR(20) | — | Да | {admin, librarian, reader} | Роль в системе |

### audit_log (Журнал аудита)

| Поле | Тип | PK/FK | Обязат. | Домен/Ограничения | Описание |
|------|-----|-------|---------|-------------------|----------|
| id | INTEGER | PK | Да | AUTO_INCREMENT | ID записи |
| user_id | INTEGER | FK → users | Да | Существует в users | Кто выполнил |
| action | VARCHAR(100) | — | Да | Не пустой | Тип операции |
| details | TEXT | — | Нет | — | Подробности |
| timestamp | DATETIME | — | Да | Автоматически | Время операции |

## Связи между сущностями (ERD)

```
books (book_code PK)
  │
  └──< loans (book_code FK) ──> readers (reader_id FK)
                                    │
users (id PK)                       │
  │                                 │
  └──< audit_log (user_id FK)      │
                                    │
                              readers (reader_id PK)
```

Тип связей: books 1:N loans, readers 1:N loans, users 1:N audit_log

## Валидации данных (≥5 правил)

| # | Правило | Тип | Применение |
|---|---------|-----|-----------|
| V-01 | Обязательные поля не пустые | Обязательность | books, readers, loans |
| V-02 | issue_date <= due_date | Диапазон дат | loans |
| V-03 | return_date >= issue_date (если задана) | Диапазон дат | loans |
| V-04 | reader_id существует в readers | Ссылочная целостность | loans |
| V-05 | book_code существует в books | Ссылочная целостность | loans |
| V-06 | Один book_code — не более одной активной выдачи (return_date=NULL) | Бизнес-правило | loans |
| V-07 | Повторная загрузка CSV не создаёт дублей (идемпотентность по PK) | Идемпотентность | импорт |
| V-08 | role ∈ {admin, librarian, reader} | Домен | users |
