## 1. Цель

Зафиксировать формальные контракты обмена данными между подсистемами АБИС: CSV-шаблоны, JSON-схемы эндпоинтов, а также DFD (диаграмма потоков данных). Документ используется при разработке и приёмке.

## 2. DFD (Data Flow Diagram)

### Уровень 0 — контекстная диаграмма

```
          ┌─────────────┐
          │Библиотекарь │──────┐
          └─────────────┘      │
                               │ форма, запросы
                               ▼
     ┌────────────┐     ┌──────────────┐     ┌────────────┐
     │ CSV-файлы  │────►│   АБИС       │────►│Отчёты CSV  │
     │(books,     │     │              │     │ (просрочки)│
     │ readers,   │     │              │     └────────────┘
     │ loans)     │     └──────┬───────┘             ▲
     └────────────┘            │                     │
                               ▼                     │
                      ┌──────────────┐               │
                      │  БД SQLite   │───────────────┘
                      └──────────────┘
                               ▲
                               │ обмен
                      ┌──────────────┐
                      │1С:Предприятие│
                      └──────────────┘
```

## 3. CSV-контракты

Все файлы: **UTF-8**, разделитель `;`, первая строка — заголовок. Кавычки обрамляют значение только при наличии в нём `;`, `"` или перевода строки.

### 3.1 `books.csv`

| Поле       | Тип     | Обязательное | Правила                                  |
|------------|---------|--------------|------------------------------------------|
| book_code  | string  | Да           | Уникальный, 1–20 символов, [A-Z0-9-]+    |
| title      | string  | Да           | 1–500 символов, без лидир./хвост. пробелов|
| author     | string  | Да           | 1–300 символов                            |
| year       | int     | Да           | 1000 ≤ year ≤ 2100                        |
| udk        | string  | Нет          | Пример: `004.65`                          |

Пример:
```
book_code;title;author;year;udk
B001;Архитектура промышленных систем;Мартин Р.;2020;004.4
```

### 3.2 `readers.csv`

| Поле        | Тип     | Обязательное | Правила                                  |
|-------------|---------|--------------|------------------------------------------|
| reader_id   | string  | Да           | Уникальный, 1–12 символов                |
| fio         | string  | Да           | 1–200 символов                            |
| group       | string  | Да           | 1–50 символов                             |

### 3.3 `loans.csv`

| Поле        | Тип     | Обязательное | Правила                                      |
|-------------|---------|--------------|----------------------------------------------|
| loan_id     | string  | Да           | Уникальный                                    |
| reader_id   | string  | Да           | FK → readers                                  |
| book_code   | string  | Да           | FK → books                                    |
| issue_date  | date    | Да           | ISO 8601 (YYYY-MM-DD)                         |
| due_date    | date    | Да           | issue_date ≤ due_date                         |
| return_date | date    | Нет          | Пусто или ≥ issue_date                        |

## 4. JSON-контракты (ключевые)

```json

{
  "reader_id": "R0042",
  "book_code": "B001",
  "issue_date": "2026-04-20",
  "due_date": "2026-05-04"
}

  "book_code": "B001",
```

## 5. Таблица маппинга (CSV → SQLite → 1С)

| Источник    | Поле        | SQLite таблица.поле      | 1С-объект.реквизит                      | Преобразование / правило       |
|-------------|-------------|--------------------------|-----------------------------------------|--------------------------------|
| books.csv   | book_code   | books.book_code (PK)     | Справочник.Книги.Код                    | TRIM, регистр неизменен        |
| books.csv   | title       | books.title              | Справочник.Книги.Наименование           | TRIM, NOT NULL                 |
| books.csv   | author      | books.author             | Справочник.Книги.Автор                  | TRIM                           |
| books.csv   | year        | books.year               | Справочник.Книги.ГодИздания             | INT, 1000–2100                 |
| books.csv   | udk         | books.udk                | Справочник.Книги.УДК                    | TRIM, опционально              |
| readers.csv | reader_id   | readers.reader_id (PK)   | Справочник.Читатели.Код                 | TRIM                           |
| readers.csv | fio         | readers.fio              | Справочник.Читатели.ФИО + Наименование  | TRIM                           |
| readers.csv | group       | readers.group_name       | Справочник.Читатели.Группа              | TRIM                           |
| loans.csv   | loan_id     | loans.loan_id (PK)       | Документ.ВыдачаКниги.Номер              | PK, идемпотентный ключ         |
| loans.csv   | reader_id   | loans.reader_id (FK)     | Документ.ВыдачаКниги.Читатель           | FK-проверка                    |
| loans.csv   | book_code   | loans.book_code (FK)     | Документ.ВыдачаКниги.Книга              | FK-проверка                    |
| loans.csv   | issue_date  | loans.issue_date         | Документ.ВыдачаКниги.Дата               | ISO → Дата 1С                  |
| loans.csv   | due_date    | loans.due_date           | Документ.ВыдачаКниги.ПлановаяДатаВозврата | ISO → Дата 1С              |
| loans.csv   | return_date | loans.return_date        | Документ.ВозвратКниги.ДатаВозврата       | ISO → Дата 1С, NULL допустим   |

## 6. Валидаторы и тест-сьют

- `data/schemas/` — JSON-Schema для всех трёх CSV.
- `tests/validators/` — Python-скрипты проверки входных файлов и golden-сверки.
- `backend/tests/test_import_idempotent.py` — проверка идемпотентности.

| Показатель                          | Целевое значение       |

| Экспорт просрочек, 100+ записей     | < 1 сек                |
