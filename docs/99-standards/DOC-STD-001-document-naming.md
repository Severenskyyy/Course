# Стандарт именования и версионирования документов — АБИС (Кейс 8)

| Версия | Статус | Дата       |

| v1.0   | Final  | 2026-03-20 |

## 1. Цель

Единое правило именования и версионирования всех документов проекта, чтобы любой член команды (и преподаватель) легко находил нужный артефакт.

## 2. Шаблон имени файла

```
DOC-<CAT>-NNN-<slug>.md
```

- `DOC` — префикс (все документы команды).
- `CAT` — трёхбуквенная категория.
- `NNN` — порядковый номер в категории, трёхзначный.
- `slug` — короткое человекочитаемое имя (kebab-case, английские буквы).

## 4. Версионирование

В шапке каждого документа:

```
| Версия | Статус | Дата       |
|--------|--------|------------|
| v1.0   | Final  | 2026-04-25 |
```

Значения статуса:
- `Draft` — в работе, может измениться.
- `Review` — отправлен на ревью.
- `Final` — утверждён, изменения только через PR с обновлением версии.

Версия инкрементируется как:
- major: перераспределение структуры или смена смысла;
- minor: добавление разделов;
- patch: опечатки, форматирование.

## 5. Правила ссылок

- Ссылаться на документы относительным путём из текущей папки: `../05-data-and-integration/DOC-INT-002-....md`.
- Не ссылаться на конкретную строку/заголовок при возможности избежать.

## 6. Правила для изображений

- Изображения хранить рядом с документом в папке `image/`.
- Имя файла: `DOC-CAT-NNN-figure-N.png`.
- Любая диаграмма должна иметь текстовое описание на случай недоступности изображения.
