## 2. Проблема и обоснование

Учебная библиотека ведёт учёт книг, читателей и выдач в Excel-таблицах. На практике это приводит к:

- невидимым просрочкам — нет «дашборда» по выданным экземплярам;
- ручному двойному вводу — каталог 1С и Excel-журнал не связаны;
- ошибкам в датах и человеческому фактору при формировании отчётов;
- невозможности быстро ответить на вопрос «у кого сейчас на руках книга X».

АБИС закрывает эти пробелы единой системой с веб-интерфейсом библиотекаря и интеграцией с 1С через файловый CSV-обмен.

## 6. Границы проекта (in/out scope)

**В рамках:**

- Web-интерфейс библиотекаря на React 18 + TypeScript + Vite (9 экранов).
- REST API на FastAPI с JWT-авторизацией, 8 роутерами, SQLite.
- 1С-конфигурация (учебная редакция): справочники «Книги»/«Читатели», документы «ВыдачаКниги»/«ВозвратКниги», регистр сведений «Выдачи», отчёт СКД «Просрочки».
- Двусторонний CSV-обмен между API и 1С (UTF-8 BOM, разделитель `;`).
- Тестовый набор T1: 250 книг / 150 читателей / 600 выдач.
- Дашборд KPI и фильтрованный отчёт «Просрочки».
- Три роли: `admin`, `librarian`, `reader`.
- Полный пакет документации согласно DOC-STD-001 и дорожной карте ОПД.

**Вне рамок (по согласованию с куратором):**

- Мобильное приложение.
- SSO / интеграция с Active Directory.
- Биллинг штрафов и пеней.
- Online-обмен с 1С (REST/SOAP) — в MVP только CSV.
- Реальные персональные данные (используется синтетический набор T1).

## 7. KPI проекта

| № | Показатель | Формула | Целевой порог |
|---|---|---|---|
| KPI-1 | Доля просроченных выдач | (просрочки / активные) × 100% | ≤ 15% |
| KPI-2 | Скорость отчёта «Просрочки» | Время ответа API на 600 строк | < 3 секунд |
| KPI-3 | Идемпотентность импорта | Кол-во дублей после повторного CSV | 0 |
| KPI-4 | Покрытие критичных эндпоинтов pytest-ами | Доля FR с автотестом | 100% |
| KPI-5 | Расхождения между API и 1С после CSV-обмена | Кол-во отличающихся записей | 0 |

KPI-1 рассчитывается на лету на эндпоинте `/api/reports/kpi`. KPI-2 замеряется при прогоне отчёта на наборе T1 (600 выдач). KPI-3 покрыт тестом `test_csv_data_exists` + ручной регрессией.

## 9. Допущения

- Учебная редакция 1С:Предприятие 8.3 доступна на машинах команды и куратора.
- SQLite приемлем как СУБД для учебного проекта (согласовано с куратором).
- Обмен с 1С — асинхронный, через CSV; реальное время не требуется.
- Аутентификация — локальная (JWT HS256), без интеграции с университетской СЗИ.
- Демо проводится локально (`localhost`) или с одного ноутбука команды.

## 13. Критерии успеха проекта

- Все четыре защиты (H1–H4) пройдены без переноса.
- Рабочая сборка v2.2-lite демонстрирует end-to-end сценарий: логин → каталог → выдача → возврат → дашборд → 1С-обмен.
- Полный пакет документации согласно дорожной карте ОПД.
- 17/17 pytest passing на момент финальной защиты.
- 1С-конфигурация интегрирована: CSV-обмен в обе стороны, отчёт СКД работает.
- Внутренние стандарты команды (DOC-STD-001) применены ко всем документам.
