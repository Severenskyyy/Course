## 2. Экраны и их состав

### Экран 1. LoginPage

```
+--------------------------------------------+
|                                            |
|              📚 АБИС                       |
|     Учебная библиотека · Кейс 8           |
|                                            |
|     Логин:    [______________]            |
|     Пароль:   [______________]            |
|                                            |
|              [   Войти   ]                |
|                                            |
|     Демо: admin / admin2026               |
|           librarian / library2026         |
|           reader / reader2026             |
+--------------------------------------------+
```

- Поля: `username`, `password`.
- Действие «Войти» → POST `/api/auth/login` → JWT в `localStorage`.
- Ошибка 401 → красный баннер «Неверный логин или пароль».

+----------------------------------------------------------+
| Главная                                                  |

| │ 250  │ │ 150  │ │ 600  │ │ 87 (15%)│                  |

| • Выдача BK0042 → RD0017 (сегодня 10:24)                |

- Лента «последние действия» из `audit_log` (последние 10).
- Запросы: `GET /api/reports/kpi`, `GET /api/reports/audit?limit=10`.

## 3. Логика навигации (state-machine, упрощённо)

```
Anonymous ─► /login ─► [JWT] ─► Authenticated
Authenticated:
  role=admin     ─► все экраны
  role=librarian ─► Home/Catalog/Readers/Loans/Overdue/Profile
  role=reader    ─► Home (read-only)/Catalog (read-only)/Profile
Logout ─► Anonymous
```

Защищённые роуты реализованы через React `RoleGuard` в `src/components/layout/`.

---

## 4. Принципы UI

- Минимум кликов: основные операции — за 1–3 клика от HomePage.
- Подсветка просрочек — везде красным, активных в зоне риска — жёлтым.
- Все таблицы поддерживают сортировку по колонкам и сквозной поиск.
- Подтверждение опасных действий (удаление, возврат с просрочкой) — через диалог.
- Локализация — русский, формат дат `DD.MM.YYYY` в UI, `YYYY-MM-DD` в API.
