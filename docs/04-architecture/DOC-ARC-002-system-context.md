| Версия | Статус | Дата       |

## 1. Назначение

Документ фиксирует: что есть «система АБИС», кто её пользователи, с какими внешними сущностями она взаимодействует и где проходит граница решения. Используется как вход для архитектуры и ТЗ.

## 2. Диаграмма системного контекста (C4-level 1)

```
            ┌────────────────┐
            │  Библиотекарь  │
            └───────┬────────┘
                    │ операции выдач/возвратов
                    ▼
┌─────────────────────────────────────────────────┐
│           СИСТЕМА АБИС «Библиотека»             │
│                                                 │
│   Frontend (React)  ◄──►  Backend (FastAPI)     │
│                              ▲                  │
│                              │ JSON/CSV         │
│                              ▼                  │
│                      1С:Предприятие 8.3         │
└─────────────────────────────────────────────────┘
                    ▲                    ▲
                    │                    │
                    │ справочники CSV    │ отчёты CSV
         ┌──────────┴────┐      ┌────────┴────────┐
         │  Администратор │      │ ФОИВ/Руководство│
         └────────────────┘      └─────────────────┘
                    ▲
                    │ (только просмотр своих выдач, Could)
         ┌──────────┴────┐
         │    Читатель    │
         └────────────────┘
```

## 3. Внутренние компоненты (C4-level 2, Container)

| Компонент          | Технология                   | Ответственность                                              |
|--------------------|------------------------------|--------------------------------------------------------------|
| Web UI             | React 18 + TypeScript + Vite | Единая точка взаимодействия для всех ролей                   |
| Backend API        | FastAPI + SQLAlchemy         | Бизнес-логика, авторизация, CRUD-операции, отчёты            |
| БД                 | SQLite 3                     | Хранение каталога, читателей, выдач, пользователей, аудита   |
| 1С-подсистема      | 1С:Предприятие 8.3           | Документооборот, справочники, регистр «Выдачи», отчёт 1С     |
| Слой обмена        | CSV / JSON                   | Импорт/экспорт между backend и 1С                            |

## 4. Внешние акторы и интерфейсы

| Актор             | Тип             | Как взаимодействует с системой                       | Формат обмена |
|-------------------|-----------------|------------------------------------------------------|---------------|
| Библиотекарь      | Первичный       | Web UI, роль «Библиотекарь»                          | HTTPS         |
| Администратор     | Первичный       | Web UI + прямой доступ к 1С                          | HTTPS / RDP   |
| Читатель (Could)  | Вторичный       | Web UI, роль «Читатель», только просмотр своих выдач | HTTPS         |
| Руководство (ФОИВ)| Внешний         | Получает отчёты как CSV/PDF                          | Email / файл  |

## 5. Что **входит** в границы решения

- Web-интерфейс для всех ролей.
- REST API со всеми CRUD-операциями и отчётами.
- БД с перечнем сущностей (books, readers, loans, users, audit_log).
- Импорт CSV и экспорт CSV.
- 1С-конфигурация со справочниками, документами, регистром и отчётами.
- Синтетический датасет T1 для демо.
- Документация, BPMN, ERD, ТЗ.

## 6. Что **не входит** (Won’t)

- Интеграции с внешними промышленными АБИС (ИРБИС, МАРС).
- Электронный каталог с полнотекстовым поиском.
- Модуль штрафов и денежных расчётов.
- Интеграция с мобильными push-уведомлениями.
- Выгрузка статистики в общероссийские системы (АРБИКОН).
- Резервное копирование «out of the box» (возлагается на эксплуатацию).

## 7. Нефункциональные требования верхнего уровня

| NFR                      | Значение                                   | Комментарий                                   |
|--------------------------|--------------------------------------------|-----------------------------------------------|
| Доступность              | Один рабочий день                          | Учебный проект — без SLA                      |
| Производительность       | Отчёт по 100+ записям < 1 с                | Проверено на T1 (≥100)                        |
| Надёжность               | Идемпотентный импорт                       | Повторная загрузка не создаёт дублей          |
| Безопасность             | JWT + bcrypt, RBAC                         | 3 роли с разграничением                       |
| Локализация              | Русский язык                               | Технические термины — в бэк-коде на английском|

## 8. Архитектурные принципы

1. **Контракт впереди кода.** Сначала OpenAPI и ERD, потом реализация.
2. **Один владелец артефакта.** Каждый документ/модуль имеет единственного ответственного.
3. **Идемпотентность обмена.** Все операции импорта безопасны к повторам.
4. **Разделение слоёв.** UI ↔ API ↔ БД; 1С не общается с БД напрямую, только через API или CSV.
5. **Минимум внешних зависимостей.** SQLite, а не полноценный СУБД-сервер; CDN-библиотеки — фиксированные версии.
